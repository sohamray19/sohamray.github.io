      Particularly, I built a real time, streaming, ETL (extract-transfer-load) data pipeline for data processing and analysis. The idea of the project was to build a modular pipeline that was able to ingest data from any source and process it swiftly, and reliably. In the time I spent at Exadatum, I built a modular pipeline that was able to ingest streaming data from Twitter, using the Twitter API and Apache Flume (a data connector).Once I had the data, I was feeding it to an Apache Kafka producer. To do this, I fed individual tweets to the python tensorflow model using a Restful service. Keep in mind, the idea of the project was to build a scalable pipeline, so the pluggable sentiment analysis model was just one of many possible ML models for the pipeline. I also built a module to simply store the data into a Apache Hive table.

      Next, to make the model readily deployable, we used Maven as a build tool to not only ingest the data using Flume and consume it from Kafka using Spark, but also to trigger the ML model written in Python and activate the RESTful service all at once.

  • Machine Learning

    I am deeply interested in machine learning algorithms, specifically NLP. I am also interested in scaling these algorithms to fit large datasets.
    This is what I've done so far:

    • Cybage Software: At Cybage, I was a part of the Decision Mines team, the machine learning division. Here, I built a feedback analysis system that can analyse sentiment of text, irrespective of domain.

      It proved especially challenging to abstract the domain, since most text classification tasks are heavily dependent on context. To tackle this problem, I combined the Yelp, Amazon and Hotel Reviews datasets available online and purged the domain dependent words among them. Next, I wrote a data preprocessing pipeline to clean, preprocess and enforce labelling on the data. However, this took tremendously long to preprocess, so I used python multiprocessing to speed up the process. Finally, for sentiment analysis, I used a deep learning model, with word embeddings, and LSTMs. For hyperparameter tuning, I used Bayesian Optimization to achieve an optimal result, in minimum time.

      Our final model improved classification accuracy by 11% on the inplace system, and scored an accuracy of 80% on data from un-trained domains, that is book data, kitchen appliance review data and banking data.

      For this project, I worked with python and used the Tensorflow, NLTK and scipy libraries.
